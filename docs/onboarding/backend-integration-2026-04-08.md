이 문서는 현재 가이드가 아니라 2026-04-08 전후 백엔드 연동 작업의 이력을 남기는 참고 문서입니다.  
최신 구조와 API는 반드시 아래 문서를 우선 기준으로 확인합니다.

- [docs/api/api.md](../api/api.md)
- [docs/database/schema.md](../database/schema.md)

## 당시 작업 요약

1. 예약 모델에 회의록 관련 필드 추가
2. 회의록 잠금 API 추가
3. 사용자/예약/회의록 연동을 프런트에서 실제 백엔드 기준으로 전환
4. 회의실 테이블과 관리자/라벨 기능 추가

## 현재 시점에서 달라진 점

- 당시 문서에서 설명한 구조는 현재 저장소 구조와 일부 다릅니다.
- 현재 `infra`는 `_repo`, `_gateway` suffix 없이 대상 중심 모듈명을 사용합니다.
- 사용자 API는 관리자 생성/비활성화/관리자 권한 변경/AI 사용량 조회까지 포함합니다.
- AI 전사/회의록 초안 기능과 전사 AI 사용량 집계가 추가되었습니다.
- 배포 문서는 현재 [docs/deploy/](../deploy/README.md) 아래 문서를 기준으로 관리합니다.

## 보존 이유

- 초기 연동 범위와 배경을 추적할 때 참고할 수 있다.
- 프런트-백엔드가 어떤 시점부터 실제 API로 연결되었는지 확인할 수 있다.

운영/개발 판단 기준으로는 이 문서보다 최신 명세 문서를 우선합니다.
